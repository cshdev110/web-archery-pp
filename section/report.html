Introduction

The page 'Report' aims to explain the components of the current website and how the technology managed was used to build it, however not in-depth. Therefore, the objective is to show the significant points of each component, which are in the content panel. Hence, two technologies were implemented to create the site (HTML and CSS), making it a usable front-end. However, it requires more work to be a fully functional website, like adding dynamic capabilities using JavaScript and some back-end technologies.

Some parts of the website that were developed using CSS, as requirements, are not precisely to be written only in CSS, like some animation and shifting images, how it is shown on this website. The best practice is making use of JavaScript. However, as it was not allowed for this work to use other technologies, those features were built using only CSS, where JavaScript would be a better option, instead. Perhaps, those changes could be done in feature updates.

Structure

Globally, the website is constituted of a central storage folder, where inside are all the files needed to make it works. The main folder contains five sub-folders: docs, icons, img, sections, style, and the index.html file.

  • In the docs folder are important files that the company wants to show to give more information, like, in this case, PDF files.
  • In the "icons" folder are the icons used over each page.
  • All the pictures are in the "img" folder.
  • The section's folder stores each site page, unlike the index.html page, which is always placed in the main folder.
  • Style's folder contains all CSS files that transform a page to a specific "style".
  • The index.html is the principal file that shows up at first when a site is visited.

There are seven web pages: home, Try Archery, Membership, Disciplines, Coaching, Info, and Report. Every page has a standard CSS stylesheet file and a particular one. First, there are nine CSS stylesheets, seven of which are the particular ones, one per page and with the same name as its respective page. Those are to style the "main" section of the HTML body. The two reminded CSS stylesheets are style.css and header-footer.css. Every page links the style.css, and style.css imports the others CSS stylesheets. The header-footer.css gives the standard style for the header and footer on each page.

The style.css file is linked on each page, then imports all the other CSS stylesheets and the font used on this website. Additionally, it adds some previous global styles.

As the HTML flows, the CSS files follow it too, which means the written CSS code starts from the top to the bottom, styling the bar menu first and then the rest of the document down to the last part of the footer. Each CSS file is conformed into two parts. The first is to manage the screen with a size up to 1920px, and the second is for screens up to 720px. The mentioned makes the website responsive to fit into different sorts of devices. The first part is the most extended section of the CSS files making it with the capability to allow the site to be responsive as well. The second part is the shortest and contains less code. It is easy to differentiate every section of the CSS stylesheet files, even the HTML files, as they are separated by a long comment as a label.

Nav

Nav is the HTML tag used to build a bar menu like the one on this site located on top of it. It means the way a user can navigate the website. Although it uses only CSS and HTML, this is one of the components to be leveraged using JavaScript. It is also fully responsive; however, the best practice to make it is to use JavaScript.

For instance, the responsive bar menu uses a check input HTML tag to open it. It is a way to substitute for an event in JavaScrip. Similar to the HTML and CSS buttons in this report.
